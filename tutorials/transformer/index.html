第 00 章 · OVERVIEW

总览：一台预测机器

剥掉所有神秘感，Transformer 干的事只有一件：读入一串 token，输出「下一个 token 是什么」的概率分布。写诗、翻译、写代码、聊天，全是这一个动作反复执行的结果。

核心想法 ②

第 01 章 · TOKENS & EMBEDDINGS

分词与嵌入：从文字到数字

模型不认识汉字，只认识数字。进门要过两道手续：分词（查一本固定的「词表」，把文本切成 token，每个 token 有一个编号）和嵌入（凭编号领取一根代表它的向量）。

{{ tok }}
真实世界是怎么切的：BPE 子词
上面的演示按「字」切，真实模型用 BPE（字节对编码）：从字节开始，反复把语料中最常一起出现的相邻片段合并成新 token。结果是——高频词整个是一个 token（「今天」），生僻词被拆成几段（「魑魅魍魉」→ 4 段甚至更多）。词表大小是门权衡：切得细 → 词表小但序列长；切得粗 → 序列短但词表爆炸。主流模型词表约 5 万～25 万。

编号本身没有含义（「猫」是 3721 还是 8964 无所谓）。真正的表示是嵌入向量：一张巨大的查询表 E，第 i 行就是 token i 的向量。这些向量是训练出来的，训练完成后，语义相近的词，向量方向也相近：

{{ emb }}
xi = E[ ti ]  ,  E ∈ ℝV×d
V = 词表大小，d = 模型维度。GPT-3：V=50257, d=12288 → 光这张嵌入表就有 6.2 亿参数
小结
▸ 分词 = 查固定词表切段编号；嵌入 = 凭编号领向量
▸ 向量是学出来的，几何关系（方向、距离）承载语义
▸ 从此整句话 = 一个 n×d 的矩阵 X，后面全部是对 X 的运算
第 02 章 · POSITIONAL ENCODING

位置编码：顺序从哪来

这里有个尴尬的事实：下一章的注意力机制对所有词一视同仁——它天生看不见顺序。对它来说「猫咬了狗」和「狗咬了猫」是同一袋词。顺序必须被明确地「注入」进去。

最经典的方案（原论文）：给每个位置生成一根固定的位置向量 Ppos，直接加到词向量上。它由一组频率递减的正弦/余弦波构成——快波区分相邻位置，慢波区分远近段落，像「个位、十位、百位」的多尺度刻度尺：

{{ pe }}
PE(pos, 2i) = sin( pos / 100002i/d )   PE(pos, 2i+1) = cos( pos / 100002i/d )
输入模型的最终表示：xi = E[ti] + Pi —— 词义与位置直接相加，混进同一根向量
正弦式（2017）
固定公式、不用学，理论上可外推到没见过的长度。
学习式（GPT-2）
每个位置一根可训练向量，简单直接，但超出训练长度就没有编码可用。
RoPE 旋转式（现代主流）
不加向量，而是按位置「旋转」Q 和 K，让注意力分数天然只依赖相对距离。Llama、Qwen、DeepSeek 都用它。
小结
▸ 注意力本身不分先后，顺序信息必须显式注入
▸ 多尺度正弦波 = 给每个位置一个独一无二、且能体现相对距离的「坐标」
▸ 现代模型多用 RoPE，思想不变：让模型感知「谁在谁前面、隔多远」
第 03 章 · ATTENTION ★ 全书核心

注意力：一场加权的信息聚会

回到我们的句子：「小明喂了猫，它很开心。」——「它」指谁？你几乎瞬间锁定「猫」。注意力机制就是把这个「按相关性去别的词那里取信息」的动作，变成可微分的数学。

每个 token 从自己的向量派生出三张牌：

Q · Query
「我在找什么？」
「它」的 Q ≈ 我需要一个前文提到的动物
K · Key
「我是什么？」
「猫」的 K ≈ 我是名词、动物、被喂的对象
V · Value
「我能提供什么？」
真正被搬走的信息内容

拿自己的 Q 去和每个词的 K 做点积——方向越对得上，分数越高；分数经 softmax 变成权重；再按权重把大家的 V 加权平均搬回来。点一个词试试：

{{ arcs }}
一个精准的类比：图书馆检索
Q 是你输入的搜索词，K 是每本书的书脊标签，V 是书的内容。区别在于：注意力不是取最匹配的那一本，而是把所有书按匹配度加权混合——所以整个过程可导、可训练。

亲手算一遍

用一个小到能口算的例子：5 个 token、d=4、dk=2。下面是完整的四步，每一步的矩阵都摆在桌面上（数字是真实算出来的）：

{{ steps }}
Attention(Q, K, V) = softmax (QKT√dk) · V
Q,K ∈ ℝn×dk　V ∈ ℝn×dv　QKT ∈ ℝn×n（n² 的来源！）　输出 ∈ ℝn×dv

为什么要除以 √dk？

维度越高，点积的数值天然越大（dk 个随机项相加，方差 ∝ dk）。分数一大，softmax 就会变成「赢家通吃」——权重挤成 one-hot，梯度几乎为零，训练卡死。除以 √dk 正好把方差拉回 1。亲手拖拖看：

{{ scale }}
# 自注意力（单头）—— 全部家当就这几行
Q = X @ W_Q        # [n, d] @ [d, dk] -> [n, dk]
K = X @ W_K        # [n, dk]
V = X @ W_V        # [n, dv]
S = Q @ K.T / sqrt(dk)      # [n, n] 两两相关性
A = softmax(S, dim=-1)      # 每行归一化成权重
out = A @ V                 # [n, dv] 加权搬运信息
小结
▸ 注意力 = 用 Q·K 打相关性分数 → softmax 变权重 → 加权平均 V
▸ 每个词的新表示都是「按需混合了全句信息」的版本，「它」从此含有「猫」
▸ n×n 的分数矩阵是长文本变慢变贵的根源；√dk 是为了梯度健康
第 04 章 · MULTI-HEAD

多头注意力：同时开几场会

一次注意力只能表达一种关系模式——但「它」既要找指代对象（猫），也关心邻近的字、句法上的搭配。解决办法简单粗暴：把 d 维切成 h 份，每份配一套独立的 WQ/WK/WV，各开各的会，最后把结果拼回来。

下面是同一句话在 4 个头里的注意力矩阵（行 = 谁在看，列 = 看谁，越亮权重越大）。切换看看每个头「性格」有多不同：

{{ heads }}
headi = Attention(XWQ(i), XWK(i), XWV(i))   MultiHead = Concat(head1..h) WO
dk = d / h（例：d=768, h=12 → 每头 64 维）→ 总计算量与单个大头相同，表达力却翻倍
真实模型里的头都在干嘛？
可视化研究（BertViz 等）发现：有的头专盯上一个词，有的头追指代链，有的头总看标点，还有不少头几乎躺平全看首 token（「attention sink」现象）。没人给它们分工——全是训练自己长出来的。GPT-3 每层 96 个头，共 96 层。
小结
▸ 多头 = 在 h 个低维子空间里并行做注意力，各学一种关系
▸ Concat + WO 把各头见闻汇总回 d 维
▸ 分工不是设计出来的，是训练涌现的
第 05 章 · THE BLOCK

积木块：注意力 + 前馈网络

注意力负责「交流」——词与词之间搬运信息；它身后还跟着一个前馈网络 FFN，负责「消化」——每个位置独立地把刚收到的信息加工一遍。两者用残差连接和 LayerNorm 粘起来，就是一个标准积木块：

{{ block }}
残差连接：默认直通
输出 = x + 子层(x)。每一层只需学「要往 x 上加点什么修正」，而不是重新发明 x。梯度也能沿着这条高速公路直达底层——没有它，几十层根本训不动。
LayerNorm：音量校准
把每个位置的向量拉回「均值 0、方差 1」再进子层，防止数值越叠越飘。现代模型多用更简的 RMSNorm，且放在子层之前（Pre-LN），训练更稳。右下实验可亲手归一化一次 →

FFN 本身极其朴素：升维 4 倍 → GELU 激活 → 降回来，且对每个位置独立同样地执行（词与词互不通信）。别小看它——模型约 2/3 的参数都在 FFN 里，主流观点认为事实性知识主要存在这儿。

{{ ln }}
# 一个 Transformer 块（Pre-LN 版，现代标准）
x = x + MultiHeadAttention( LayerNorm(x) )   # 交流：token 间搬信息
x = x + FFN( LayerNorm(x) )                  # 消化：逐位置加工

# 其中 FFN（d -> 4d -> d）
FFN(x) = W2 @ gelu(W1 @ x + b1) + b2

# 整个模型 = 把这个块叠 N 次
for block in blocks:   # GPT-2: N=12/24/48, GPT-3: N=96
    x = block(x)
小结
▸ 一块 = 交流（注意力）+ 消化（FFN），残差与归一化保驾护航
▸ 堆 N 块：底层看局部搭配，越往上表示越抽象（短语 → 指代 → 语义 → 任务）
▸ 参数大头在 FFN；「知识」多存于此
第 06 章 · TRAINING

训练：文本自己就是题库

不需要任何人工标注。拿一句话，把它右移一位就是标准答案：读到「小明」该猜「喂」，读到「小明喂了猫，它很」该猜「开心」。互联网上的每一句话都自动成为无数道练习题——这叫自监督学习。

输入
小明喂了猫，它很
目标 ↓
喂了猫，它很开心
同一句话，7 个位置 = 7 道题，同时练

但有个作弊漏洞：算「它」的注意力时，全句都在桌面上——它能直接看到答案「很」。所以要盖上因果掩码：每个位置只许看自己和左边，未来一律遮住（分数设为 −∞，softmax 后权重恰好为 0）：

{{ mask }}

掩码带来一个天大的好处：训练时所有位置的预测可以一次前向全部算完，不用像 RNN 那样一个字一个字排队。这才是 Transformer 取代 RNN、吃下全部算力红利的真正原因：

{{ parallel }}

「猜得准不准」用交叉熵衡量：只看你分给正确答案的概率 p，损失 = −log p。给对的词的概率越小，惩罚增长越狠——拖动下面的滑杆感受这条曲线的脾气：

{{ loss }}
L = − (1/n) Σi log pθ( ti+1 | t1..i )
对每个位置：正确的下一个词分到的概率取 −log，再平均。训练 = 用梯度下降把它推到最低
预训练之后还有两步（现代 LLM 流水线）
① 预训练：万亿 token 上猜下一个词 → 得到「什么都见过」的基座模型；② 指令微调 SFT：在「问题-优质回答」对上继续训练 → 学会听话；③ 人类反馈强化学习 RLHF：人类给回答排序，训练它偏向被喜欢的风格。架构从头到尾没变，变的只是数据和目标。
小结
▸ 目标只有一个：最小化「下一个词」的交叉熵
▸ 因果掩码防作弊，还让训练完全并行——算力红利全靠它
▸ ChatGPT 式助手 = 同一架构 + 三阶段数据（预训练 / SFT / RLHF）

第 08 章 · THE FAMILY

家族与全景

原论文是「编码器 + 解码器」的翻译架构，后来家族按用途分了三支：

ENCODER-ONLY
BERT 系 · 理解
双向注意力（无掩码），训练靠「完形填空」。适合分类、检索、抽取。如今的 embedding 模型多属此支。
DECODER-ONLY ★
GPT 系 · 生成
因果掩码 + 猜下一个词——本教程讲的正是它。GPT、Claude、Llama、Qwen、DeepSeek……当代 LLM 几乎一统于此。
ENCODER–DECODER
T5 系 · 转换
编码器读原文，解码器边看它（交叉注意力）边写译文。翻译、摘要的经典形态，也是原论文的样子。

同一套积木，还搭出了语言之外的一切：

ViT · 把图当句子读
图像切成 16×16 的小块，每块当一个 token 排队进同一个 Transformer。视觉从此告别卷积垄断；多模态模型把图文 token 混进同一个序列。
MoE · 专家分诊
把 FFN 换成几十个「专家」，路由器每个 token 只叫醒 1–2 个。参数总量巨大、每步计算稀疏——Mixtral、DeepSeek-V3 等走这条路。
长上下文工程
n² 注意力是瓶颈：FlashAttention 重排显存访问，RoPE 外推 + 滑窗 / 稀疏注意力把上下文从 2K 推到百万级。
缩放定律 Scaling Laws
损失随参数、数据、算力按幂律平滑下降，可预测。「继续变大就会继续变好」——这条经验曲线是过去几年 AI 竞赛的总剧本。
