第 00 章 · TL;DR

十秒版：VLA 的天花板，WAM 的起点

VLA 从网页图文继承了「语义」：它能执行「把可乐罐移到 Taylor Swift 照片旁」——认得出照片，接得上学过的「移动」技能。但让它「解鞋带」就完了：这个动作没在机器人数据里出现过，语义先验帮不上运动。

DreamZero 换了个底座：用看过整个互联网视频的视频扩散模型起家。视频模型「见过」无数物体怎么被推、被倒、被折叠——物理动力学已经在权重里。剩下的事：让它顺便把动作也生成出来。

主张 ①
预测未来 = 隐式规划
联合生成「未来画面 + 动作」，动作学习从死记模仿变成逆动力学——对着自己想象的未来对齐动作。新任务 2 倍于 SOTA VLA。
主张 ②
14B 视频模型能实时控制
从 5.7 秒/块优化到 150 毫秒——38 倍加速，7Hz 闭环控制。视频扩散第一次跑进实时控制回路。
主张 ③
视频就是通用语
别的机器人或人类的「纯视频」（无动作标签）10–20 分钟就能提升 42%；30 分钟玩耍数据适配一台全新机器人。

▸ 名字故意不叫「视频动作模型」：未来的世界建模目标可以是触觉、力反馈、隐空间
第 02 章 · ARCHITECTURE ★ 论文核心

联合去噪：画面与动作一起从噪声里长出来

DreamZero 把「预测未来 H 帧画面」和「预测未来 H 步动作」写成同一个流匹配去噪问题：两团噪声共用一个去噪时间表，在同一个 14B DiT 骨干里一起变清晰。因为是一起去噪，画面和动作天然对齐——动作就是「让世界变成那个画面」的操作。拖动去噪进度：

{{ denoise }}
π( ol:l+H , al:l+H | o0:l , c , ql ) = π( ol:l+H | 历史, 指令 ) · π( al:l+H | 全部画面 )
视频预测 × 逆动力学 IDM —— 但不拆成两个模型，一个模型端到端学联合分布
zt = t·z₁ + (1−t)·z₀ ， at = t·a₁ + (1−t)·a₀ —— 视频与动作共享时间步 t，损失 = 联合速度场回归
为什么自回归、按块生成
不是一口气生成整段视频，而是像 LLM 吐 token 一样一块一块地吐（每块 = K 帧 + 对应动作，训练用 teacher forcing）。好处：KV cache 加速、能吃任意长历史、保住原生帧率让画面-动作严格对齐。消融证实：自回归比双向结构动作更顺滑。
闭环妙招：用现实覆盖梦境
自回归视频生成的死穴是误差滚雪球——越梦越歪。但机器人是闭环的：每执行完一块动作，真实相机画面就把 KV cache 里「想象的帧」替换掉。梦境每 1.6 秒被现实校准一次，误差无从积累。这是 WAM 独有的免费午餐。

动手改了哪些零件

动什么 · 冻什么
底座 Wan2.1-I2V-14B。冻结文本编码器、图像编码器、VAE；全量更新所有 DiT 块 + 新加的状态/动作编解码器。试过 LoRA 省钱——效果不行，放弃。
多路相机怎么进
头部 + 双腕多视角拼成一张大图喂进去——不改骨干一行结构，视频模型当它是一帧普通画面。
动作与训练配置
动作 = 相对关节位置；训练前滤掉发呆片段。100K 步 × batch 128，AgiBot 与 DROID 各训一份。
# 训练一步（teacher forcing · 每块独立时间步）
z1 = VAE.encode(未来K帧)          # 画面 → 潜空间
a1 = normalize(未来H步动作)
t  = uniform(0,1)                 # Flash 训练: 视频改用 t=1−Beta(7,1)
zt = t*z1 + (1-t)*noise_z         # 流匹配线性加噪（见 Transformer 篇 ch7）
at = t*a1 + (1-t)*noise_a
v  = DiT_14B([zt, at]; 干净历史块, 指令c, 本体状态q, t)
loss = ‖ v − ([z1,a1] − [noise_z,noise_a]) ‖²   # 回归联合速度场

# 推理循环（异步 · 每块 1.6s）
for 每个动作块:
    梦境帧, 动作块 = 去噪(噪声; KV_cache, c, q)   # Flash 后只需 1 步
    机器人.执行(动作块)                            # 48 步 @30Hz，GPU 同时算下一块
    KV_cache[想象的帧] = 真实相机帧                # 现实覆盖梦境，误差清零
小结
▸ 一个 14B DiT 同时去噪画面与动作，共享时间步 → 天然对齐
▸ 自回归按块吐未来，KV cache 提速 + 任意长历史 + 原生帧率
▸ 闭环回填真实观测：自回归视频最怕的误差累积被现实抹掉
第 03 章 · REAL-TIME ENGINEERING

反应鸿沟：5.7 秒 → 150 毫秒

视频扩散模型天生是个慢性子：16 步迭代去噪 × 14B 参数，朴素实现一块动作要 5.7 秒——机器人早撞墙了。第一步解法是异步执行：手上执行着上一块动作（48 步 @30Hz = 1.6 秒），GPU 同时在算下一块。只要推理比 1.6 秒快，机器人就永不停顿。拖动推理延迟感受这个约束：

最后一道工序：动作抛光
少步去噪的动作难免带高频毛刺。出厂前先把动作块上采样 2 倍 → Savitzky-Golay 滤波 → 降回原采样率——机械臂拿到手的是一条平滑曲线，不是锯齿。
小结
▸ 异步执行把约束从「算完才能动」松成「1.6 秒内算完就行」
▸ 系统级（CFG 并行、DiT 缓存）+ 实现级（编译、量化、内核）≈ 16×
▸ 剩下的 2.3× 靠改训练本身——Flash，下一章
第 04 章 · DREAMZERO-FLASH

Flash：让动作先醒，画面继续做梦

最后的瓶颈是去噪步数本身。直接砍步数会翻车：视频没去干净的噪声会污染动作。Flash 的洞察——其实我们只需要干净的「动作」，画面模糊点无所谓。于是训练时故意把两个时间表拆开：视频常驻高噪声区（Beta 分布偏置），动作照常均匀采样——逼模型学会「从模糊的梦里提取清醒的动作」：

{{ flash }}
小结
▸ 标准训练：视频/动作同步去噪 → 少步推理时动作被残余视觉噪声拖累
▸ Flash 训练：视频偏向纯噪声（E[t]≈0.125），动作均匀 → 训练态 = 推理态
▸ 去噪 4 步 → 1 步，350ms → 150ms，性能几乎无损；外加 Savitzky-Golay 滤波抹平动作毛刺
第 05 章 · DATA PHILOSOPHY

数据哲学：要「干活」，不要「排练」

传统 VLA 的数据像排练：同一个任务反复演示几十遍，模型靠重复背下状态→动作映射。DreamZero 反其道：500 小时数据来自 22 个真实场所（家庭、餐馆、超市、咖啡店、办公室），操作员连续干真实的活——平均一条轨迹 4.4 分钟、约 42 个子任务，几乎不重复。

为什么 WAM 吃得下这种「乱」数据？因为世界建模目标从每一对相邻帧里都能学到东西——不重复恰恰意味着信息密度高。消融实验：同样 500 小时，多样性数据完胜重复演示：

{{ datamix }}
一条轨迹长什么样
不是「抓一次积木」的 15 秒片段，而是「进厨房→开冰箱→取物→擦台面→摆盘」的连续作业：导航、躯干升降、双臂操作混在一条流里——正因为要伺候真实场所的货架高低与房间布局。
可复现性：DROID + 开源
自采数据发布前，先在公开的 DROID（Franka 单臂、最杂的开源机器人数据集之一）上复训验证了整套结论，并开源权重、推理代码与 PolaRiS / Genie Sim 3.0 评测脚本——不是「仅供参观」的论文。
小结
▸ 世界建模目标让「每对相邻帧」都成为监督——不重复 = 高信息密度
▸ 数据采集从「排练任务」转向「记录劳动」，成本结构完全不同
▸ 结论在自采 AgiBot 数据和公开 DROID 上双重成立
第 06 章 · RESULTS

实验：默认考「双重陌生」

评测哲学延续 π0.5 又更激进：默认设定就是陌生环境 + 陌生物体（训练与评测在不同地理位置），再分「见过的任务」和「没见过的任务」。基线是 GR00T N1.6 和 π0.5，喂完全相同的 500 小时数据、同等算力预算——纯粹比架构。

「任务」按运动 + 物体类型定义粒度：叠过红衬衫、考叠黑衬衫算「见过」；考叠袜子（运动不同）算「没见过」。10 个见过的任务（易/难抓放、堆叠、擦拭、折叠）× 4 台机器人 × 不同环境不同物体 × 8 次 = 每个模型 80 次实测。切换标签页：

第 07 章 · DISCUSSION

诚实的边界，和范式之争

代价与短板
▸ 14B 扩散模型，实时推理要双 GPU + 顶级硬件（完整 38× 需 GB200）
▸ 需要记忆的任务未评测（虽然架构天然带历史）
▸ 预训练仍是单本体，多本体联训留作未来
▸ 策略上限被视频生成质量锁死——梦不清楚就做不对
它真正改变的认知
▸ 运动泛化的钥匙在视频先验里，不在语言先验里
▸ 「提升机器人 = 提升视频生成」——两个领域从此绑定
▸ 数据可以「乱采」：干活数据 > 排练数据
▸ 纯视频（无动作标签）就能教机器人——数据金矿一下子大了几个数量级
